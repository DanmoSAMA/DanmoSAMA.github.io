单例模式

单例模式：创建全局独一无二的一个变量

写法符合职责分离原则，其中 Create 类负责创建结点，ProxySingletonCreate 类负责确保该结点的唯一性

ProxySingletonCreate 并未采用 Class 的写法，因为这样对每个实例对象而言，instance 都是不同的，因此还是要采用闭包的写法，这样第一次得到的 instance 值就会被保存下来，第二次就可以直接将它返回

还有一点，函数的返回值要指明:void，否则会报错，虽然我觉得这样有点奇怪…
